# Transcripts — Index

Meeting notes, call transcripts, decision logs. Time-stamped record of conversations that informed product or engineering choices but don't live in the code.

## When to read

- A feature has unclear motivation in code — check transcripts for the "why"
- Before re-litigating a decision — confirm it wasn't already settled
- Onboarding a contributor — historical context

## When to write

- After any stakeholder call that produced a decision or constraint
- After internal sync that changed direction on a feature
- Anytime a verbal decision affects code that's not obvious from the diff

## Conventions

- Filename: `YYYY-MM-DD_short-topic.md` (e.g. `2026-05-07_effect-migration-kickoff.md`)
- One file per session — do not concatenate
- Top of file: date, attendees, one-line summary
- Bottom: `## Decisions` and `## Action items` (with owner + date)
- Link any resulting feature memory: `See .brain/features/<slug>.md`
