# Emails — Index

Archived email threads, stakeholder communication, and external correspondence that materially affects engineering decisions or product scope.

> This folder is for **archived correspondence**, not template code. There is no email-sending code in this repo today (no Resend / SES / etc. integration). If one is added, document it under [`../rules/services.md`](../rules/services.md).

## When to read

- A constraint in code traces back to a partner / vendor / legal email
- Before responding to a stakeholder thread — pull prior context
- Auditing a feature decision driven by external communication

## When to write

- After receiving an email that imposes a constraint, deadline, or requirement
- After sending an email that commits the team to a deliverable
- When a thread changes scope or priority of in-flight work

## Conventions

- Filename: `YYYY-MM-DD_sender_topic.md` (e.g. `2026-04-12_legal_session-token-policy.md`)
- One file per thread — append replies chronologically
- Top: date received, sender, recipient(s), one-line summary
- Strip secrets (API keys, passwords, PII) — replace with `[REDACTED]`
- Link any resulting feature memory or transcript
