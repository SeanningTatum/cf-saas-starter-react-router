# i18n

**Library**: remix-i18next + i18next + react-i18next.
**URL strategy**: `/:lng/` prefix on public marketing/auth routes (`/`, `/login`, `/sign-up`); `/dashboard` and `/admin` are **not** locale-prefixed and rely on client-side i18n only.

## Usage

```typescript
// Component
import { useTranslation } from "react-i18next";
const { t } = useTranslation("admin");
return <h1>{t("dashboard.total_users")}</h1>;

// Loader / meta
import { i18nServer } from "@/i18n/i18n.server";
const t = await i18nServer.getFixedT(request, "admin");
```

A route opts into a namespace via the `handle` export:

```typescript
export const handle = { i18n: ["admin"] };
```

## Conventions

- **Namespaces** (must match `namespaces` const): `common`, `auth`, `admin`, `dashboard`, `home`, `validation`
- **Keys**: dot-notation snake_case (`dashboard.total_users`)
- **Interpolation**: `"Hello, {{name}}"` → `t("greeting", { name })`

## Adding a new language

1. Create `app/locales/{lng}/*.json` files mirroring every namespace JSON in `app/locales/en/`.
2. Add the locale code to `supportedLngs` in `app/i18n/i18n.ts`.
3. Add a label in `app/components/language-switcher.tsx` (`languageLabels[lng]`).
4. Add a `date-fns` mapping in `app/lib/date-utils.ts` (`localeMap[lng] = …`).
5. Hard-refresh `/`, click the switcher, verify both directions swap live.

## Live language toggle (no full reload)

[`app/components/language-switcher.tsx`](../../app/components/language-switcher.tsx) submits to [`app/routes/api/set-locale.ts`](../../app/routes/api/set-locale.ts) via `useFetcher().submit()`. The action writes the locale cookie via `localeCookie.serialize(lng)` (base64-JSON encoding owned by `createCookie` — see footnote). After the action returns, React Router automatically revalidates the root loader, which re-detects the locale via `i18nServer.getLocale(request)` and returns the new `loaderData.locale`. The existing `useChangeLanguage(loaderData.locale)` hook in [`app/root.tsx`](../../app/root.tsx) then flips `i18next.language`, react-i18next re-renders the tree, and the page swaps copy with no full reload.

> **Anti-pattern**: do **not** call `i18n.changeLanguage(lng)` directly from the switcher. It races with `useChangeLanguage`'s revert-on-mismatch effect in `root.tsx` and the language flips back. Drive the swap through the action + revalidation instead.

> **Cookie format**: React Router's `createCookie` base64-encodes a JSON-stringified value (e.g. `"zh"` → `InpoIg==`). Manually setting `document.cookie = "i18next=zh"` won't be parsed by `i18nServer.getLocale`. Always go through an action that calls `localeCookie.serialize`.
